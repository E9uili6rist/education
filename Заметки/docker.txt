docker run -d --name postgres -e POSTGRES_PASSWORD=whatislove --network=postgres_network postgres (запустить контейнер postgres и поключить его к сети)
docker run -d --name server_container --network postgres_network server (запустить контейнер приложения и присоединить к сети)

В контексте команд Docker -d является сокращением от --detach и означает, что контейнер будет запущен в фоновом режиме (detached mode).
Когда вы запускаете контейнер без этого флага, Docker выполняет контейнер в связанном (attached) режиме по умолчанию.
Это означает, что вывод контейнера (stdout и stderr) будет выводиться прямо в терминал, и терминал будет заблокирован до тех пор, пока контейнер не остановится.
С флагом -d Docker запускает контейнер в фоновом режиме, отсоединяясь от него после запуска.
Это позволяет терминалу остаться активным для ввода новых команд, а контейнер продолжает работать в фоновом режиме.
Использование -d особенно полезно для запуска длительно работающих контейнеров, таких как веб-серверы, серверы баз данных или системы мониторинга, которые должны работать постоянно в фоновом режиме.
